Python pip更换国内源

python语言是需要我们经常会使用 工具去安装第三方包。如果我们直接使用pip install packageName命令，是从国外的服务器去下载并安装包第三方包，下载速度是非常的慢。为了解决下载速度的问题，我们可以更换为国内的服务器进行下载。

pip国内的一些镜像

  • 阿里云: http://mirrors.aliyun.com/pypi/simple/
  • 中国科技大学: https://pypi.mirrors.ustc.edu.cn/simple/
  • 豆瓣(douban): http://pypi.douban.com/simple/
  • 清华大学 https://pypi.tuna.tsinghua.edu.cn/simple/

修改源方法

可以在使用pip的时候在后面加上-i参数，指定pip源
eg: pip install scrapy -i https://pypi.tuna.tsinghua.edu.cn/simple

Anaconda

简介

官方网站: https://www.anaconda.com

Anaconda是什么？

  • Anaconda是一个数据科学开发平台, 支持 Linux, Mac, Windows系统
  • 为Python提供了与的功能，可以很方便地解决多版本python并存、切换以及各种第三方包安装问题
    • 包管理与pip的使用类似
    • 环境管理则允许用户方便地安装不同版本的python并可以快速切换

Anaconda的优势是什么？

  • 预装了命令来进行package和environment的管理
    • conda的设计理念——conda将几乎所有的工具、第三方包都当做package对待，甚至包括python和conda自身！因此，conda打破了包管理与环境管理的约束，能非常方便地安装各种版本python、各种package并方便地切换。
  • 预装了某个版本的python、众多packages
  • 预装了多个应用程序
    • Anaconda Navigtor ：用于管理工具包和环境的图形用户界面，在命令行模式下输入的众多管理命令也可以在Navigator中手动实现。
    • Jupyter notebook ：基于web的交互式计算环境，可以编辑易于人们阅读的文档，用于展示数据分析的过程。
    • qtconsole ：一个可执行 IPython 的仿终端图形界面程序，相比 Python Shell 界面，qtconsole 可以直接显示代码生成的图形，实现多行代码输入执行，以及内置许多有用的功能和函数。
    • spyder ：一个使用Python语言、跨平台的、科学运算集成开发环境。
    • …

Conda的环境管理

在命令行中如何创建自己的虚拟环境？

  • 为什么要为每一个独立的项目都创建一个单独的虚拟环境？

    • 独立——相互隔离，互不影响
    • 纯净——只有一个项目用的包和依赖，好管理
    • 方便——摒弃频繁安装/卸载包和依赖

  • 创建一个名称为的虚拟环境并指定python版本为3.7(这里conda会自动找3.7中最新的版本下载)

    自定义的learn环境除了python自带的一些官方包之外是没有其他包的, 是一个比较干净的环境。

Anaconda所谓的创建虚拟环境其实就是安装了一个真实的python环境, 只不过我们可以通过activate,conda等命令去随意的切换我们当前的python环境, 用不同版本的解释器和不同的包环境去运行python脚本.
